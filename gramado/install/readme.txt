install/

    Ao invés de inicializar o ambiente do usuário,
    como o gdeshell ou gui, então inicializaremos esse ambiente
    de instalação do sistema.

    isso será usado pela versão live para instalar o sistema definitivamente
    no disco alvo, que pode não ser o mesmo ao qual 
    esse sistema deu boot.

    como o sistema de instalação ainda não tem uma interface gráfica
   então ele terá se proprio shell igual o gde shell,
   será um shell de instalação.   ishell.bin

   #importante:
   O ishell.bin chamará outros aplicativos nessa pasta
   para realizar as tarefas envolvendo a instalação.
   assim como poderá chamar outros apps do sistema.
